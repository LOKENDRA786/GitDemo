Fast-forward Merge

A fast-forward merge happens when there is a direct path between the branch you're on and the branch you're merging into. In this case, Git moves the pointer forward, and no new commit is created.


# Scenario: Branch 'feature' is ahead of 'main'
git checkout main
git merge feature

No Fast-forward Merge

A no fast-forward merge is used to ensure that a merge commit is always created, even when a fast-forward merge is possible. This is useful for preserving the history of branches.


# Scenario: Force Git to create a merge commit
git checkout main
git merge --no-ff feature

Three-way Merge

In a three-way merge, there is no direct path between the branches, and Git must create a new commit that ties the histories of both branches together.


# Scenario: 'feature' and 'main' have diverged
git checkout main
git merge feature

Merge Conflicts

Merge conflicts occur when two branches modify the same part of a file in different ways. Git cannot automatically resolve these conflicts and requires manual intervention.


# Scenario: Resolve conflicts manually
git checkout main
git merge feature

# When conflict arises, resolve it in the files and then
git add [resolved-file]
git commit

Merge Unrelated Histories

This happens when you try to merge branches that have completely different commit histories, like merging two separate projects. Git requires the --allow-unrelated-histories flag.


# Scenario: Merging branches with different histories
git checkout main
git merge --allow-unrelated-histories feature
